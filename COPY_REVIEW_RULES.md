# コピー確認ルール

このプロジェクトでは、ユーザーが読む文章を勝手に変更しない。

デザイン、UI、アニメーション、レイアウト、実装については、Codexの判断で改善してよい。

ただし、以下は必ず事前に確認する。

- 見出し
- サブコピー
- 説明文
- CTA
- ボタン文言
- バナー前の文章
- リザルト画面の文章
- プロフィール文
- その他ユーザーが読む文章すべて

文章を変更したい場合は、以下を提示し、ユーザーが選択または修正した後に実装する。

- 現在の文章
- 変更理由
- 改善案 3〜5案

## 確認フォーマット

```text
【場所】

例：
リザルト画面 メインコピー

【現在】

思ってたより、
いい条件で見てもらえそうです。

【改善したい理由】

・少しAIっぽい
・人が言わない表現
・もっと自然にしたい

【改善案】

① ○○○

② ○○○

③ ○○○

④ 自由入力
```

## 特に確認が必要な箇所

- トップコピー
- ローディング文言
- 結果画面
- 年収カード
- ポイント見出し
- サービス紹介前の文章
- カードタイトル
- ボタン
- フッター
- プロフィール文
- Threads誘導文

## Codexの役割

Codexは、UIデザイナー、フロントエンドエンジニア、UXデザイナーとして提案する。

コピーライターとして勝手に文章を決めない。

コピーは必ずユーザーと一緒に決める。ユーザーがOKを出した文章だけを実装する。
